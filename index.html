Goals

  • Monitor plant condition using sensors (soil + environment).
  • Detect abnormal states using thresholds and rules.
  • Provide clear status output (display/logging/dashboard).
  • Justify value creation using ROI / company-value approach (course method).

Problem statement

Plant care is often reactive: people notice problems too late (overwatering, dryness, low light). The system should continuously collect data, reduce uncertainty, and support timely decisions.

System contradictions (from systems analysis)

  • Need more control vs limited attention of the user.
  • Better accuracy vs cost and complexity of sensors.
  • Continuous monitoring vs energy / maintenance constraints.
  • More data vs need for simple interpretation (clear status, alerts).

Scope & assumptions

  • Educational prototype with realistic architecture.
  • Threshold-based logic with future upgrade path to ML.
  • Focus on reliability, readability, and reproducibility.
  • All results documented and reproducible from repository.

System architecture

Data flow: sensors → microcontroller → processing (rules/thresholds) → output (display/log/dashboard) → user decision.

LayerRoleExamples in this project
Sensing Collect raw signals Temperature/Humidity, Light, Soil moisture, etc.
Processing Convert signals into indicators Filtering, calibration, thresholds, state machine
Decision support Interpret indicators OK / Warning / Critical + recommendations
Output Show status to user OLED/LCD, Serial log, optional web dashboard

Key metrics

Indicators used to evaluate the system and demonstrate value.

  • Detection accuracy (events captured)
  • Time to detect abnormal state
  • System uptime / stability
  • User effort reduction

Implementation note: start with rule-based logic, then improve using better sensors and calibration.
